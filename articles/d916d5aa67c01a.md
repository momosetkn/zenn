# Vite で react-doc-viewer の PDF 表示を react-pdf に差し替えて、日本語の文字化けを避ける

@cyntler/react-doc-viewer は便利ですが、PDF によっては日本語がうまく表示されないことがあります。

このプロジェクトでは、ドキュメント一覧やヘッダー周りは @cyntler/react-doc-viewer を使いながら、PDF の描画だけを react-pdf に差し替える構成にしました。

この記事では、Vite 環境でその構成を作る手順と、実際に詰まったポイントをまとめます。

## この記事でやること

- Vite + React + TypeScript の環境で @cyntler/react-doc-viewer を使う
- PDF レンダラだけ react-pdf に差し替える
- pdfjs-dist の worker と補助アセットを Vite で扱う
- @cyntler/react-doc-viewer の既定 loader と react-pdf の相性問題を避ける

## 動機

@cyntler/react-doc-viewer はファイル種別ごとにレンダラを差し替えられます。

今回は次の方針を取りました。

- ドキュメント表示コンポーネント本体は @cyntler/react-doc-viewer
- PDF の実描画は react-pdf
- 日本語表示のために cmaps と standard_fonts を pdfjs-dist から配布

## 依存関係

```typescript
{
  "dependencies": {
    "@cyntler/react-doc-viewer": "^1.17.1",
    "pdfjs-dist": "4.8.69",
    "react": "^18.3.1",
    "react-dom": "^18.3.1",
    "react-pdf": "^9.1.1"
  }
}
```

ここで大事なのは、pdfjs-dist のバージョンを react-pdf が使うものに合わせることです。

react-pdf 側が 4.8.69 を使っているのに、アプリ側の pdfjs-dist が 4.10.x になると、次のようなエラーが出ます。

Warning: UnknownErrorException: The API version "4.8.69" does not match the Worker version "4.10.38".

worker を pdfjs-dist/build/pdf.worker.min.mjs?url から import している場合、ここがずれると表示できません。

## Vite で worker を読む

```typescript
react-pdf は内部で pdf.js を使うので、worker の URL を設定します。

import { Document, Page, pdfjs } from 'react-pdf';
import 'react-pdf/dist/Page/AnnotationLayer.css';
import 'react-pdf/dist/Page/TextLayer.css';
import pdfWorkerSrc from 'pdfjs-dist/build/pdf.worker.min.mjs?url';

pdfjs.GlobalWorkerOptions.workerSrc = pdfWorkerSrc;
```

Vite では ?url を付けると、worker ファイルを URL 文字列として import できます。

## 日本語表示のための補助アセット

日本語を含む PDF をきれいに表示するため、cmaps と standard_fonts を配布します。

package.json では次のように public/pdfjs へコピーしています。
```typescript
{
"scripts": {
"copy-statics": "mkdir -p ./public/pdfjs && cp -r ./node_modules/pdfjs-dist/cmaps ./public/pdfjs && cp -r ./node_modules/pdfjs-dist/standard_fonts ./public/pdfjs",
"postinstall": "pnpm copy-statics"
}
}
```

そして Document にオプションを渡します。

```typescript
const pdfOptions = {
cMapUrl: '/pdfjs/cmaps/',
cMapPacked: true,
standardFontDataUrl: '/pdfjs/standard_fonts/',
};
```

## react-doc-viewer から PDF だけ差し替える

react-doc-viewer には複数の renderer が入っているので、標準の PDFRenderer を外して、自前の PdfRenderer を先頭に入れます。

```typescript
import DocViewer, {
DocViewerRenderers,
PDFRenderer,
} from '@cyntler/react-doc-viewer';

const pluginRenderers = [
PdfRenderer,
...DocViewerRenderers.filter((renderer) => renderer !== PDFRenderer),
];
```

これで PDF のときだけ自前の renderer が選ばれます。

## 実際の PdfRenderer

ポイントは 3 つあります。

1. react-pdf に uri をそのまま渡す
2. 表示幅に合わせて Page の width を更新する
3. @cyntler/react-doc-viewer 側の既定先読み loader を止める

概略はこんな形です。

```
import { useEffect, useMemo, useRef, useState } from 'react';
import type { DocRenderer } from '@cyntler/react-doc-viewer';
import { PDFRenderer as DefaultPDFRenderer } from '@cyntler/react-doc-viewer';
import { Document, Page, pdfjs } from 'react-pdf';
import 'react-pdf/dist/Page/AnnotationLayer.css';
import 'react-pdf/dist/Page/TextLayer.css';
import pdfWorkerSrc from 'pdfjs-dist/build/pdf.worker.min.mjs?url';

pdfjs.GlobalWorkerOptions.workerSrc = pdfWorkerSrc;

const pdfOptions = {
cMapUrl: '/pdfjs/cmaps/',
cMapPacked: true,
standardFontDataUrl: '/pdfjs/standard_fonts/',
};

// react-pdf に uri をそのまま読ませるため、
// react-doc-viewer 側の既定先読みは行わない。
const passthroughFileLoader: NonNullable<DocRenderer['fileLoader']> = ({
fileLoaderComplete,
}) => {
fileLoaderComplete();
};

const PdfRenderer: DocRenderer = ({ mainState }) => {
const containerRef = useRef<HTMLDivElement>(null);
const [containerWidth, setContainerWidth] = useState<number>();
const [numPages, setNumPages] = useState(0);
const [scale, setScale] = useState(1);
const file = useMemo(() => mainState.currentDocument?.uri, [
mainState.currentDocument?.uri,
]);

  useEffect(() => {
    const container = containerRef.current;
    if (!container) return;

    const updateWidth = () => {
    setContainerWidth(Math.max(container.clientWidth - 48, 0));
    };

    updateWidth();

    const resizeObserver = new ResizeObserver(updateWidth);
    resizeObserver.observe(container);

    return () => resizeObserver.disconnect();
  }, []);

  useEffect(() => {
    setNumPages(0);
    setScale(1);
  }, [file]);

  const pageNumbers = useMemo(
    () => Array.from({ length: numPages }, (_, index) => index + 1),
    [numPages]
  );

  if (!file) return null;

  return (
    <div ref={containerRef}>
    <Document
      file={file}
      options={pdfOptions}
      onLoadSuccess={({ numPages: nextNumPages }) => setNumPages(nextNumPages)}
    >
      {pageNumbers.map((pageNumber) => (
      <Page
        key={pageNumber}
        pageNumber={pageNumber}
        width={containerWidth || undefined}
        scale={scale}
        renderAnnotationLayer
        renderTextLayer
      />
      ))}
    </Document>
    </div>
  );
};
```

PdfRenderer.fileTypes = DefaultPDFRenderer.fileTypes;
PdfRenderer.weight = DefaultPDFRenderer.weight;
PdfRenderer.fileLoader = passthroughFileLoader;

## なぜ fileLoader を止めるのか

PdfRenderer.fileLoader = DefaultPDFRenderer.fileLoader; にしても表示自体は動く場合があります。ただしその場合でも裏では react-doc-viewer 側の先読みが走るので、役割分担としては中途半端です。

react-pdf に uri をそのまま渡す構成にするなら、明示的に no-op loader にしておいた方が意図がはっきりします。

## ヘッダーやダウンロードボタンは @cyntler/react-doc-viewer 側で持つ

PDF の描画は差し替えても、ヘッダー UI は @cyntler/react-doc-viewer 側の override を使えます。

```typescript
config={{
header: {
disableHeader: false,
overrideComponent: headerOverride,
},
}}
```

これでファイル名表示やダウンロードボタンをまとめて管理できます。

## まとめ

@cyntler/react-doc-viewer と react-pdf を組み合わせると、ドキュメントビューア全体の使い勝手を保ちながら、PDF だけ描画品質を調整できます。

今回のポイントは次の 3 つでした。

- pdfjs-dist のバージョンを react-pdf と揃える
- cmaps と standard_fonts を配布する
- @cyntler/react-doc-viewer の既定 loader と react-pdf の役割を混ぜない

PDF の日本語表示で困っていて、でもビューア全体は @cyntler/react-doc-viewer を使いたい、というケースではかなり扱いやすい構成だと思います。
